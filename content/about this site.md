## 不是简介的简介

这里是互联网上的一个普通网站，基于 [Quartz 4](https://quartz.jzhao.xyz/) 构建，数据位于本地和 GitHub 仓库，使用赛博大善人 Cloudflare 部署。本地使用 [[obsidian|Obsidian]] 编辑草稿并推送至 GitHub。

网站的建立完全出于作者的兴趣，没有明确目的，也没有结束之日，可能在十年后，也可能在明天。作者原本的 [WordPress 博客](https://imzm.im)于 2026年4月9日 归档，目的是为了节省维护多个网站的精力。本网站留言板基于 Giscus，需要一个 GitHub 账号，由此带来的不便敬请原谅，但我可能不会改。

*20260503更新*

在 PC 端，网站首页左侧是文章列表，有若干文件夹按主题区分，其余均放在根目录下。左上角的标题下方分别是搜索框、暗黑模式和阅读模式按钮，点击阅读模式会隐藏网页两个侧边栏。右侧分别是当前页面的标题目录和反向链接列表，点击标题或者反向链接可以快速跳转到相应位置。每个页面下方都有一个 Giscus 留言板。底部是版权信息。网站尽可能按照我的意愿连接了所有页面，不排除可能有些许遗漏。Quartz 4 默认带有网状知识图谱 [Graph View](https://quartz.jzhao.xyz/features/graph-view)，经过权衡，我决定移除这一板块。

网站在 Quartz 4 的基础上做了少许 CSS 修改，以符合我个人的审美。

这个网站可以是一个数字花园、博客、个人维基，或者 Web 2.0 时代常常使用的个人网站一词。但我想这些描述都是标签，一旦选用其中一个，似乎就被框定了。我愿意其不被任何标签框定住。我曾经有数个网站，最终收敛至一个，希望它能代表我想要在互联网上记录和表达的主要文本容器。

## 心路历程

使用互联网二十余年，留下的痕迹近年来才逐渐变多。早年注册各种博客服务商，大多随着时间渐长消失不见，我对遗失的内容有深深的不安感。尽管我生产的这些内容，都是对他人没什么价值的东西，但对我个人有充分价值，因此想要尽可能留下一些痕迹。当前的架构，本地不同设备里有一份，GitHub 上有一份，大大缓解了我的焦虑。我的私人 Obsidian 笔记库亦如是。

用 WordPress 写博客十年后，我还尝试过建立不同版本的数字花园。例如 [[tiddlywiki|TiddlyWiki]] 和 DokuWiki，但因种种原因没有延续。TiddlyWiki 使用自己的 wiki 方言，首次加载缓慢；DokuWiki 过于文档化与多用户合作的形式，都不太符合我对一个网站的要求。直到发现 Quartz，才意识到原来存在一种静态网站生成器，在本地写完内容，通过推送转换为 HTML 发表，流畅且优雅，并天然解决了副本的问题。像这样的工具还有 Astro、Hugo、Jekyll、Eleventy 等等，但我先入为主，选择了 Quartz，再也没有变化过。

2025年4月1日，我用 Quartz 4 建立了这个全新的站点。看一眼仓库，我竟然已经 commit 了~~1800多次~~ 180多次。我很享受频繁更新内容的时光。当然，目前还没有 star，唯一一个 watch 还是自己给的。有两位朋友不辞劳苦给我留下两条评论，不胜感激。

我离开 WordPress 还有一个重要原因。除了软件本身越来越臃肿、漏洞越来越多以外，WordPress 的托管提供商 WP Engine 和 WordPress 创始人 Matt Mullenweg 以及他的 WordPress.org 和 WordPress.com (Automattic) 在2025年底又有[大量纷争](https://techcrunch.com/2025/01/12/wordpress-vs-wp-engine-drama-explained/)，社区不再纯粹，用户不胜其烦。开源精神在被商业社会的浸染下也会变得面目可憎。从那时候开始，WordPress 在我眼里不再眉清目秀，对于在 WordPress 编辑器里写东西的欲望越来越小，甚至还有一些排斥。

Markdown 似乎也要焕发第二春了。除了有些脑子有问题的 KOL 以外，一般人都会觉得这是一种不错的语法，在大多数场景下都适用。进入2026年以来，各类 Agent 工具爆发，几乎所有 Agent 都读 Markdown，而不是 json 或者其他语言。这是人类和人工智能之间的最大公约数，因此继续用 Markdown 写文章是自然而然的选择。

想到哪写到哪，到这里草稿也应该收尾了。谢谢你看到这里。

本站评论区使用 [Giscus](https://giscus.app/)，评论功能基于 GitHub Discussions。评论区需要登录 GitHub 账户后方可留言。加载评论区时，浏览器会连接 Giscus/GitHub；如果你登录 GitHub 并留言，你的 GitHub 账号信息、评论内容和相关交互将由 GitHub/Giscus 按其规则处理，评论内容将默认公开显示。

本站由 Cloudflare 部署和加速。和大多数网站一样，Cloudflare 可能会在提供网络服务、安全防护和访问加速时处理必要的访问日志，例如 IP 地址、User-Agent、请求时间等基础网络信息。

本站的暗黑模式、目录展开状态等偏好设置可能会保存在你浏览器的 Local Storage 本地存储中。这些信息只保留在你的设备上，不会由本站主动发送到任何服务器。站内搜索在浏览器本地完成，搜索关键词或任何内容不会提交给服务器。

如果你不希望留下 GitHub 相关信息，可以不使用评论功能；如果你不希望浏览器保留本地偏好，可以清除本站的浏览器数据或使用匿名模式访问。

简言之，我没有兴趣知道你是谁，我只想大概了解一下有多少人访问了我的网站。
